# 블로그를 만들게 된 계기

프로젝트를 진행할 때, 코드를 작성하면서 새로운 것들을 공부하게 됩니다.  
하지만 제가 공부하고 이해한 내용을 타인에게 말로써 알리는 것은 어려운 일입니다.

또한, 공부했던 내용을 제가 5년, 10년 후에도 기억하고 있을지는 알 수 없습니다.  
기록을 해 놓으면 나중에 같은 내용을 보았을 때 더 빨리 이해할 수 있을 거 같습니다.

이런 이유들로 블로그를 개설하게 되었습니다.


# Gibhub Pages를 선택한 이유

velog, 티스토리, Medium 등 다양한 블로그 플랫폼 중에서도 Github Pages를 선택한 이유는 다음과 같습니다.


### 완전한 커스터마이징

**Github Pages**는 **HTML**, **CSS**, **JavaScript**를 직접 작성하고 수정할 수 있기 때문에 블로그의 디자인, 레이아웃, 기능까지 완전히 제어할 수 있습니다.  


### 자체 호스팅 및 도메인 설정

**Github Pages**는 자체 호스팅을 지원하고, 자신의 도메인을 연결할 수 있습니다.  
예를 들어, ***rakblog.com*** 처럼 나만의 도메인을 사용할 수 있습니다.


### 개인 공부

장점인 완전한 커스터마이징의 단점은 코드를 작성할 줄 알아야 기능을 구현할 수 있다라는 것입니다.
블로그를 만드는 과정은 곧 스스로 성장할 수 있는 기회라고 생각해서 **Github Pages**를 선택했습니다.

이제 Github 블로그를 만들어보겠습니다.
